`flutter pub get` must run before `pod install` because the Podfile reads `FLUTTER_ROOT` from `Flutter/Generated.xcconfig` (and raises if missing). The local native engine lives at `../native` and is consumed through the Podfile path reference; after adding it, `pod install` resolves both the vendored `qwen_echo_engine (0.1.0)` and the git commit of the underlying llama.cpp source.
