The module is split by concern rather than layer: `engine_core` owns the C entry points and lifecycle; `pipeline_stages` provides the three real-time processing nodes; `inference_backend` wraps llama.cpp as the single inference provider consumed by all stages; `shared_utilities` supplies audio capture, model loading, and offline policy; `platform_hal` abstracts OS-specific audio/thermal/memory/thread/NPU calls behind a stable C ABI; `monitoring_and_tracing` polls those HALs and pushes metrics back through the same `native_port` channel used by stages. Cross-child wiring happens at startup in `engine_manager.cpp`, which constructs the stage graph, binds each stage's input/output queues, and registers the FFI bridge — so no child links directly to another child's implementation, only to the headers under `include/`.
