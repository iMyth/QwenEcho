Flat collection of self-contained `test_*.cpp` files under `native/tests/`, each a standalone executable with its own `main()`. Two testing styles coexist: RapidCheck-driven property tests (`rc::check` / `RC_ASSERT`) in most files, and a hand-rolled `run_test` harness with a global pass counter in `test_asr_stage.cpp`. Each test file targets one production header from `native/src/` (e.g. `asr_stage.h`, `model_loader.h`, `sentence_segmenter.h`, `engine_manager.h`, `audio_collector.h`, `echo_types.h`, `native_port.h`). External platform dependencies are isolated by providing `extern "C"` mock implementations linked into the test binary — notably `hal_audio_capture_*`, `hal_thread_set_realtime_priority`, and `native_port_post_sample_drop` in `test_audio_collector.cpp` replace real HAL/native-port code so the tests run without hardware or OS-specific libraries.
