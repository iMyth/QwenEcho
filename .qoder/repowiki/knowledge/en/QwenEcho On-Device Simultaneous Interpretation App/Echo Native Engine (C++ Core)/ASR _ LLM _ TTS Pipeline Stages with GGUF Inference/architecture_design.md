Each file implements one stage of the ASR→LLM→TTS pipeline as an opaque `XxxStage` struct plus an `extern "C"` public API (`*_create`, `*_destroy`, `*_set_thermal_mode`, and a processing entry point). A single `std::thread` runs a worker loop per stage: ASR uses a mutex+condition_variable queue to pull `LockedSegment`s, while LLM and TTS poll lock-free `BoundedSPSCQueue`s. All three stages depend on a common `gguf_inference.h` wrapper (`gguf_inference_create/gguf_inference_generate/gguf_inference_destroy`) that encapsulates llama.cpp; each stage holds a `GgufContext*` and falls back to in-process stub generation when the model path is absent or loading fails. Cross-stage communication flows through typed bounded queues (`AsrToLlmElement`, `LlmToTtsElement`) and `native_port_*` messages for UI streaming and latency warnings. Thermal throttling is coordinated via an atomic `throttle_mode` flag — ASR resamples 16k→8kHz, LLM shrinks its sliding context window from 512 to 256 tokens, and TTS keeps its own SLA budget. The dependency direction is strictly inward: stages → `bounded_spsc_queue.h` + `native_port.h` + `hal_accelerator.h` + `gguf_inference.h`; no cross-includes between the three stage files.
