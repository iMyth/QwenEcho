Three independent C++ leaf modules exposed via `extern "C"` entry points and consumed by the higher-level pipeline:
- `audio_collector.cpp` implements a real-time audio capture loop: it sets the calling thread to RT priority, registers a HAL callback (`hal_audio_capture_start`) that writes directly into a lock-free `AudioRingBuffer`, and detects sample drops by comparing expected vs. actual sample counts against a 10 ms threshold.
- `model_loader.cpp` owns three fixed-size `ModelSlot`s (ASR=0, LLM=1, TTS=2) backed by `mmap`ed read-only GGUF files; each slot holds an fd, mapped pointer, file size, and a placeholder `InferenceContext`. Validation walks the GGUF header KV pairs looking for `general.file_type` (mapped via `ftype_to_ggml_type`), falls back to scanning tensor descriptors for the first accepted quantization, and rejects F32/F16 or Q2_K quants through `is_accepted_quant`. The inference context is currently a stub allocating per-type heap buffers (~64/128/32 MB).
- `offline_policy.cpp` provides a single `verify_offline_policy` gate run at engine init: it checks every model path starts with the app sandbox prefix, then on Android/Linux scans `/proc/self/maps` for `libcurl.so`/`libnghttp2.so`, and on Apple uses `dlsym(RTLD_DEFAULT, ...)` to detect curl or custom telemetry symbols. No network code is compiled in — platform branches are gated by `__ANDROID__/__linux__/__APPLE__` macros.

Dependency direction is strictly inward: these files depend only on platform HAL headers (`hal_audio.h`, `hal_thread.h`, `native_port.h`) and the local `model_loader.h` / `offline_policy.h`; they do not import any higher-level stage code. The coding intent signals the current `InferenceContext` stubs will be replaced with llama.cpp ggml contexts once the submodule is integrated.
