This module is a Kiro feature specification (not source code) organized in the standard `requirements-first` workflow: `requirements.md` defines 16 user stories with acceptance criteria; `design.md` specifies the two-layer architecture (Flutter UI Shell + C/C++ Native Engine), the six-thread lock-free pipeline (Audio Collector → Ring Buffer → VAD/Sentence Segmenter → ASR → LLM → TTS), the HAL abstraction over Android NNAPI/Vulkan and iOS CoreML/Metal, the four-function Dart FFI bridge (`InitQwenEchoEngine`, `StartEchoPipeline`, `StopEchoPipeline`, `RegisterEchoMessagePort`), and 22 formal correctness properties covering ring-buffer integrity, thermal/memory state machines, and message format invariants. `tasks.md` decomposes implementation into 19 waves of tasks each tagged to specific requirements, with property-based tests (RapidCheck for C++, fast-check for Dart) interleaved as sub-tasks. `tasks.meta.json` records execution history per task including chat session IDs and timestamps, while `.config.kiro` declares `specId`, `workflowType: requirements-first`, and `specType: feature`.
