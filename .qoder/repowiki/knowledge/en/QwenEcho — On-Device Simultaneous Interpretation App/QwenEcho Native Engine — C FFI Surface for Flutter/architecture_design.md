The `native/` tree is a single CMake target (`qwen_echo_engine`, STATIC) whose public boundary is `include/ffi_bridge.h` — exactly four `extern "C"` functions (`InitQwenEchoEngine`, `StartEchoPipeline`, `StopEchoPipeline`, `RegisterEchoMessagePort`) consumed by Flutter via Dart FFI. Internally it composes five sibling concerns: `engine_core` (lifecycle + state machine), `pipeline_stages` (ASR/LLM/TTS stages with bounded SPMC queues), `platform_hal` (Android/iOS implementations selected at build time via `if(ANDROID)` in CMakeLists.txt), `audio_and_latency` (ring buffer, collector, latency tracker), and `system_and_model` (memory/thermal monitors, GGUF loader). Cross-child wiring happens through shared headers under `include/` plus the `hal/` ABI; tests under `tests/` are per-file executables linked against either the full engine or a minimal source subset, all using RapidCheck fetched via FetchContent. The same header set is also packaged as an iOS framework via `qwen_echo_engine.podspec`.
