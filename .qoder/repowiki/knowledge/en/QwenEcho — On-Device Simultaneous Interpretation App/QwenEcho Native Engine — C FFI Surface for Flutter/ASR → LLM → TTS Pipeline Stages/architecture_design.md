Each file implements one stage of the native processing pipeline as an opaque `XxxStage` struct plus a C API (`*_stage_create`, `*_stage_destroy`, optional setters). A dedicated `std::thread` runs a worker loop per stage; inter-stage data flows through `BoundedSPSCQueue<T>` elements (`AsrToLlmElement`, `LlmToTtsElement`) declared in headers outside this scope. Cross-stage communication is strictly unidirectional: ASR→LLM→TTS, with no cross-include between the three files. Each stage owns its input queue pointer (and optionally an `AcceleratorContext*` for HAL inference) but never the output queue — ownership stays with the caller who wires the chain. Inference is delegated to stub functions (`stub_infer_tokens`, `stub_translate_tokens`, `stub_tts_synthesize`) that call into `hal_accelerator.h`; real models are injected at runtime via the accelerator context. UI feedback and latency warnings are emitted exclusively through `native_port_post_*` calls, keeping stages decoupled from the host. Thermal/Throttle mode is communicated via an `atomic<int>` settable by `*_stage_set_thermal_mode`, which each stage reads at segment boundaries to adjust behavior (downsampling rate, context window size). SLA budgets are enforced per stage (200 ms first partial for ASR, 450 ms first token for LLM, 100 ms TTFA for TTS) by measuring `steady_clock` deltas against `*_LATENCY_BUDGET_MS` constants and reporting violations via `native_port_post_latency_warning`.
