Two cooperating C++ objects expose a C-style opaque-handle API:
- `engine_manager.cpp` implements `EngineManager`, a state machine over `ENGINE_UNINITIALIZED → ENGINE_INITIALIZING → ENGINE_READY → ENGINE_RUNNING → ENGINE_STOPPING → ENGINE_ERROR`. It owns a `ModelLoader` (loads ASR/LLM/TTS GGUF models via `model_loader_load`) and lazily creates a `PipelineController` on first `start_pipeline`. All transitions are serialized by a single `std::mutex`.
- `pipeline_controller.cpp` implements `PipelineController`, which wires the full cascade: `AudioCollector → AudioRingBuffer → SentenceSegmenter → AsrStage → BoundedSPSCQueue<AsrToLlmElement> → LlmStage → BoundedSPSCQueue<LlmToTtsElement> → TtsStage`, plus `ThermalMonitor`, `MemoryMonitor`, and `LatencyTracker`. Each stage runs on its own worker thread; inter-stage backpressure is provided by bounded SPSC queues. A 2-second deadline (`kStopDeadlineMs`) bounds graceful stop, polling segmenter idle + queue drain before destroying resources in reverse order.
- Dependency direction: `engine_manager` depends on both `model_loader` and `pipeline_controller`; `pipeline_controller` depends on all stage/monitor primitives and `hal_accelerator`. `engine.cpp` is a placeholder stub including only `echo_types.h`.
- Both managers use `calloc` + placement-new for their internal structs so C++ members (`std::mutex`, `std::atomic<bool>`) coexist with raw-C allocation/deallocation.
