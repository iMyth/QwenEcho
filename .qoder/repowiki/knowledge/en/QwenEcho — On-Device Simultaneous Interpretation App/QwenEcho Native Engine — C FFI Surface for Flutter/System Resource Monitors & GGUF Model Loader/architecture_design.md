Three independent C++ translation units, each exposing an opaque handle + create/start/stop/get/destroy extern "C" API:
- `memory_monitor.cpp` runs a single background thread polling `hal_memory_get_rss()` every 2 s against a platform limit, with upward-only hysteresis (OK → WARNING at 85 % → CRITICAL at 95 %) and posts `MSG_MEMORY_WARNING` via `native_port_post_memory_warning` on critical transitions.
- `thermal_monitor.cpp` runs a separate thread polling `hal_thermal_get_temperature()` every 5 s through a three-mode state machine (Normal ↔ Throttle ↔ Critical) with symmetric hysteresis thresholds (43/42 °C, 50/45 °C), notifying the UI shell via `native_port_post_thermal_state` and invoking a user callback on each transition.
- `model_loader.cpp` owns a fixed array of three `ModelSlot`s (ASR=0, LLM=1, TTS=2), opens GGUF files with `open(2)`, reads and validates the header magic, parses the first tensor's quantization type by walking the KV metadata section (`gguf_get_first_tensor_quant`), requires an INT4 variant (Q4_0/Q4_1/Q4_K), then `mmap`s the file read-only with `MADV_SEQUENTIAL` and attaches a placeholder `InferenceContext` whose `context_memory` is estimated per model type. All OS calls go through POSIX syscalls; HAL and native-port are the only cross-boundary dependencies.
