The workspace is a single Flutter project (`pubspec.yaml` in the root) whose `lib/` tree hosts the Dart application shell (dart_app) and whose `native/` tree holds the CMake-built static library (native_engine). The iOS wrapper (ios_platform) is an Xcode project that embeds both the Flutter framework and the locally built native pod. Cross-child wiring happens through two contracts: (1) a four-function C FFI surface (`InitQwenEchoEngine`, `StartEchoPipeline`, `StopEchoPipeline`, `RegisterEchoMessagePort`) declared in `native/include/ffi_bridge.h` and consumed by `lib/src/native_bridge.dart`; (2) a typed `EchoMessage` stream published via a native port and subscribed to by both the split-screen view and status overlays. Tests live under `test/` (dart_tests) and `native/tests/` (RapidCheck), while `kiro_specs/` documents the feature contract that all children implement against. There is no top-level orchestrator script; each child is built independently (`cmake --build .` for native, `flutter build` for Dart/iOS) and linked at the platform wrapper level.
