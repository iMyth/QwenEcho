Tests are organized by source package: top-level `messages_test.dart` and `split_view_test.dart` cover the core message parser (`EchoMessage.fromRawList`) and the composite SplitView/SpeakerHalf widgets; `test/ui/` mirrors `src/ui/` with one test file per component — `line_buffer_test.dart` (pure LineBuffer state machine), `text_display_test.dart` (TextDisplay rendering), `status_bar_test.dart` (thermal indicator), and `warning_overlay_test.dart` (memory/latency warnings). `widget_test.dart` is the default Flutter counter smoke test. Widget tests use `flutter_test`'s `WidgetTester`, inject dependencies via constructor parameters or `GlobalKey` state handles, and drive async streams through `dart:async` `StreamController<EchoMessage>.broadcast()` instances torn down with `addTearDown`. Pure-logic tests (LineBuffer) follow a `setUp`/`tearDown` pattern that constructs and disposes the buffer per test.
